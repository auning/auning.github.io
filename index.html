Windows11 + WSL2 + Ubuntu + PyCharm + Conda 深度学习环境搭建完整指南

在深度学习开发中，Linux环境通常比Windows环境具有更高的训练效率。但是Ubuntu对于日常使用不够友好，因此WSL2（Windows Subsystem for Linux 2）成为了一个完美的折中方案。经过测试，在WSL2中的训练效率相比Windows提升了大约25%。

本文将详细介绍如何在Windows11环境下搭建一个高效的深度学习开发环境。

📋 环境概述

技术栈组合

  • 操作系统: Windows11 + WSL2 Ubuntu
  • 开发环境: PyCharm Professional
  • 包管理: Anaconda/Conda
  • 深度学习框架: PyTorch
  • GPU加速: CUDA + cuDNN

性能对比

⚡ 第二步：配置CUDA环境

重要提示: WSL2与Windows11共用显卡驱动，因此只需安装CUDA Toolkit和cuDNN，无需安装显卡驱动。

安装CUDA Toolkit

  1. 访问 NVIDIA CUDA官网
  2. 选择CUDA 11.8版本（与PyTorch官方支持版本对应）
  3. 重要: 选择WSL2-Ubuntu版本进行安装

配置CUDA环境变量

安装cuDNN

严格来讲cuDNN不能叫安装。它其实是对CUDA的一些补充，所以”安装”过程很简单。

  1. 访问 NVIDIA cuDNN官网（这一步可能需要注册英伟达账号）
  2. 下载cuDNN 8.8.1 for CUDA 11.x版本
  3. 将文件保存到Windows环境，然后直接复制到WSL2 Ubuntu的home目录下（和在Windows环境中复制粘贴一样操作）

注意：操作要在相应的文件夹下进行！解压之后得到cuda目录，cuda目录下面有include和lib64两个子目录，将这两个目录下面的所有文件拷贝到CUDA 11.8安装路径对应的目录下面即可。

🔍 常见问题解决

CUDA相关问题

问题: nvcc -V 命令找不到
解决: 检查环境变量配置，确保CUDA路径正确

问题: PyTorch检测不到CUDA
解决: 确认安装的PyTorch版本与CUDA版本匹配

WSL2相关问题

问题: WSL2启动缓慢
解决: 调整.wslconfig配置，减少分配的资源

问题: 文件权限问题
解决: 使用chmod命令调整文件权限

PyCharm集成问题

问题: 无法连接到WSL解释器
解决: 确认WSL2正在运行，检查解释器路径是否正确
